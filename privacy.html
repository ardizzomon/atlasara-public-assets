Privacy Policy

This Privacy Policy explains how Atlasara Post Factory handles data when preparing and publishing content for Atlasara-owned brands.

Last updated: June 4, 2026

1. Information We Process

Atlasara Post Factory may process post images, captions, hashtags, product URLs, alt text, publishing metadata, platform account IDs, OAuth access tokens, board IDs, page IDs, and similar technical configuration needed to prepare or publish social media posts.

2. How Information Is Used

Information is used to generate post packages, prepare platform-specific files, authenticate approved API requests, publish content to authorized accounts, and keep a local record of generated or queued posts.

3. Storage

The tool stores generated assets and configuration in the local project folder. API credentials and tokens should be stored only in the local .env file and must not be committed to source control. Public post assets may be copied to the public site folder when needed so platforms can fetch media by HTTPS URL.

4. Sharing

Atlasara Post Factory sends data to connected platforms only as needed to create, upload, or publish authorized posts. It does not sell personal information. It does not intentionally share OAuth tokens, private session data, or passwords with third parties.

5. Browser Sessions

The tool is not designed to read browser cookies, saved passwords, or private browser session stores. Platform connections should be made through official OAuth and developer API flows.

6. Data Retention

Generated assets and local metadata remain in the project folder until deleted by an authorized operator. Publicly hosted output files remain available until removed from the public site and source repository.

7. Security

Atlasara Post Factory uses local environment variables for sensitive API credentials. Operators should keep credentials private, rotate credentials if exposed, and avoid granting broader platform permissions than needed.
